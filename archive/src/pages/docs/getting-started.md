Welcome! This guide helps you understand how to use this documentation effectively.

## Navigation Structure

This documentation is organized using the Diataxis framework, which categorizes content by purpose:

### Tutorials
Learning-oriented guides that take you through a complete project step-by-step. Perfect for newcomers who want hands-on experience.

**When to use**: You're new to a topic and want to learn by doing.

[Browse Tutorials →](/docs/guides/tutorials/)

### How-To Guides
Problem-solving guides focused on accomplishing specific tasks. Assumes you have basic knowledge.

**When to use**: You know what you want to do and need the steps.

[Browse How-To Guides →](/docs/guides/how-to/)

### Reference
Technical specifications, command references, and API documentation. Detailed and accurate information.

**When to use**: You need to look up specific details or parameters.

[Browse Reference →](/docs/reference/)

### Explanation
Conceptual content that explores topics in depth. Provides context and understanding.

**When to use**: You want to understand the "why" behind something.

[Browse Explanations →](/docs/explanation/)

## Search and Navigation

### Using Search
Press `Ctrl+K` (or `Cmd+K` on Mac) to open the search dialog. Search works across all documentation pages.

### Sidebar Navigation
The sidebar shows the complete documentation structure. Click sections to expand/collapse them.

### Table of Contents
Each page includes a table of contents on the right showing all headings for quick navigation.

## Content Standards

All documentation follows these principles:

- **Clear language**: Technical but accessible
- **Working examples**: All code examples are tested
- **Up-to-date**: Content is regularly reviewed
- **Accessible**: Designed for all readers

## Getting Help

If you can't find what you're looking for:

1. Use the search feature (`Ctrl+K`)
2. Check related topics in the sidebar
3. Visit the [portfolio site](https://kentknowsme.com) for contact information

## Contributing
